         * <a name="a3"></a>A new special outline file: scheduler.opml. <a href="#a3">#</a>
         * <a name="a4"></a>Automatically opened at startup, if it's present. <a href="#a4">#</a>
         * <a name="a5"></a>There are five defined summits: <a href="#a5">#</a>
            * <a name="a6"></a>startup <a href="#a6">#</a>
            * <a name="a7"></a>everySecond <a href="#a7">#</a>
            * <a name="a8"></a>everyMinute <a href="#a8">#</a>
            * <a name="a9"></a>everyHour <a href="#a9">#</a>
            * <a name="a10"></a>overnight <a href="#a10">#</a>
         * <a name="a11"></a>They define the times that the scripts underneath run. <a href="#a11">#</a>
         * <a name="a12"></a>Each script has a title and underneath it is the code that runs when the script runs. <a href="#a12">#</a>
         * <a name="a13"></a>Example <a href="#a13">#</a>
            * <a name="a14"></a>startup <a href="#a14">#</a>

                  * <a name="a18"></a>storage.ct = 0 <a href="#a18">#</a>
            * <a name="a19"></a>everySecond <a href="#a19">#</a>
               * <a name="a20"></a>counter <a href="#a20">#</a>
                  * <a name="a21"></a>storage.ct++; <a href="#a21">#</a>
         * <a name="a22"></a>Names are unicase, so startup is the same as StARtuP. <a href="#a22">#</a>
         * <a name="a23"></a>Comments are ignored at the top level and at the names level. Not sure if they're ignored in the scripts but they should be. <a href="#a23">#</a>
         * <a name="a24"></a>There's an item in the Special files sub-menu of the Files menu.  <a href="#a24">#</a>
         * <a name="a25"></a>You can have sections at the top level that don't match the names, they are ignored. <a href="#a25">#</a>

      * <a name="a218"></a>When it's time to specify the interface, the actual params <a href="#a218">#</a>
         * <a name="a219"></a>Look for prior art. If there's a popular app that works like yours does, the default should be do exactly what they do. Even if you don't like something about what they do. <a href="#a219">#</a>
         * <a name="a220"></a>Strive for simplicity, approachability. You want a user to look at it and nod their head, saying "this makes sense." <a href="#a220">#</a>
